Email Request Workflow

Use this guide when sending a booking request from the quote screen. The form creates a pre-filled email in your default mail client so operations can review and dispatch the load.

Receipt-style composed body sections

Selecting Compose Email builds a plain-text message with fixed sections that keep shipment data easy to scan. The generated body mirrors a receipt layout with separators and consistent labels.

  • Quote Details: Includes the quote ID and whether a return quote is requested.
  • Shipment Specifications: Lists origin, destination, pieces, weight, weight type, and accessorial labels.
  • Pricing Breakdown: Shows base charge, each accessorial amount, and the final total (plus the email admin fee when configured).
  • Logistics Contacts: Captures shipper and consignee names, addresses, contacts, phone numbers, and optional notes.

Piece count and weight method

The shipment section always includes total pieces and weight in pounds, then prints the saved weight method in parentheses:

  • Actual when the quote is priced using measured weight.
  • Dimensional when the quote is priced by dimensional weight rules.

This output comes directly from quote.weight_method, so users do not need to select it again on the email form.

Return quote checkbox behavior and intent

The highlighted return shipment section appears near the end of the form, directly above Compose Email. It includes the return quote checkbox and, when selected, additional fields for desired return pickup date, return accessorial selection, and return-specific notes.

In the composed message, the app always sets Return Quote: YES/No in Quote Details. If the box is selected, a dedicated [ RETURN SHIPMENT REQUEST ] block is added immediately after consignee notes so operations can review return handling requirements in one place.

The purpose is to alert operations that the same customer may need a quote for the reverse lane, so the team can follow up with return pricing after the outbound request is reviewed.

Mailto handoff to operations inbox

The form does not send mail directly from the web app. Instead, it creates a mailto: link that opens your local email client with the subject and body prefilled. The recipient is always the operations inbox: operations@freightservices.net.

  • Review the message content in your email client before sending.
  • Add attachments or internal recipients if your process requires them.
  • Send the message to complete the handoff from quoting to operations.
